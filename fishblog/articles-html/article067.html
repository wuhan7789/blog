1.  Java的特性



                    与其他语言相比，java语言更简单。比如c++要求程序员分配和回收内存，java不需要，java实现了内存空间的自动分配和回收，不需要也不允许程序员直接对内存进行操作，这也体现了java的安全性

java的跨平台性：​java与其他语言的最大不同就是java不是将源码程序编译成机器码，而是编译成一种称为字节码的中间代码，这种代码只有在java虚拟机上才能运行。java编译成字节码后不依赖于任何平台，无需修改就可以在任何装了java虚拟机的平台运行。

2.  面向对象的特征




继承：就是把事物相同部分提取出来组成一个父类，子类拥有父类的非私有属性和方法。

封装：把一类事物的属性和操作这些属性的行为封装成一个类，行为由这个类自己决定。

多态：类的多态：父类引用指向子类对象，有继承有重写，引用指向的具体类型和调用类型的方法只有在程序运行期间才能确定。

                    操作符的多态：俩遍都是整形那 + 号为相加，如果有一边是字符串那 + 号是字符串连接符。

抽象：把一类事物的特性行为抽象出来成为一个类。

4. 基本数据类型和包装类




因为Java面向对象语言，不建议直接操作基本数据类型（不具备面向对象特征），所以提供了基本类型的包装类对象，包装类封装了操作基本数据类型的方法。比如手动拆装箱，类型之间的转换。自动拆装箱就是编译期间自动调用包装类拆装箱的方法。

6.   String 、StringBuffer  、StringBuilder 




都是对字符串的操作

String 是不可变的字符串，底层是Final修饰的char[] 字符数组。

StringBuffer    和    StringBuilder是可变的字符串（char[] 字符数组），再增加长度的时候创建一个新的数组，把旧数据复制到新数组中这样来实现的

StringBuffer 是 线程安全的，一般涉及线程安全效率都会比较低；StringBuilder是线程不安全的。

应用场景的话：操作少量不变的字符串用String；

                                                多线程下操作大量可变字符串用StringBuffer；​

                                                 单线程下操作大量可变字符串用StringBuilder。




7.   简述集合




集合分为俩种，一种是继承Conllection接口只存value的，一种是继承Map接口存Key，Value的。

List 和 Set 接口就是继承的Conllection，然后List是有序可重复的，Set是无序不可重复的。

Map    Key不可以重复，只可以重复。相同的Key会覆盖掉原来的Value。




8.  HashMap、 HashTable 、ConcurrentHashmap

HashMap  的Key  和Value 都可以存放Null , HashTable 不可以。      HashMap不是线程安全的，效率低，HashTable是线程安全的，效率高
